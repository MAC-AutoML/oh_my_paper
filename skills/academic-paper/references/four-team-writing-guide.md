# Academic Paper writing guide

This guide keeps manuscript drafting evidence-aware and reviewer-readable.

1. Do not start from fluent prose; start from claims, evidence, and contribution shape.
2. Every central claim should map to an evidence row or an explicit caveat.
3. The outline should answer: problem, gap, idea, method, evidence, insight, limitation.
4. Abstracts should tell a complete scientific story, not list implementation details.
5. Introductions should create conflict: what progress exists and why it is insufficient.
6. Related work should classify prior work and name the closest contrast axes.
7. Method sections should explain why each design exists before giving implementation detail.
8. Experiment sections should ask what scientific question each table or figure answers.
9. Figures and tables need one takeaway, one claim link, and readable captions.
10. Limitations should be precise, scoped, and honest without destroying the contribution.
11. Style calibration may imitate structure and rhythm, not copy private text verbatim.
12. Bilingual abstracts should preserve claims exactly across languages.
13. If evidence is missing, draft a placeholder question rather than inventing a result.
14. If numbers are user-provided, preserve units, settings, and provenance.
15. If numbers are model-generated, label them as hypothetical and keep them out of final claims.
16. The writing lane cannot certify its own submission readiness.
17. A strict reviewer loop must inspect the draft before final packaging.
18. Keep revision notes separate from the main paper so the author can audit changes.
19. Do not weaken caveats merely to sound more confident.
20. Do not concede every reviewer criticism automatically; concessions need evidence.
21. The final draft should include a title, abstract, introduction, related work, method, experiments or evaluation plan, limitations, and references when applicable.
22. The writing handoff should include known weak spots for review.
23. Preserve claim IDs across section edits when possible.
24. Prefer deletion of unsupported claims over cosmetic hedging.
25. Good paper prose is a disciplined argument, not a work log.
